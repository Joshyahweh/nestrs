# Benchmark Storage Playbook

This playbook describes how to keep long-term benchmark history outside CI artifacts.

## Goals

- preserve benchmark trend data beyond short artifact retention windows
- keep raw report files and dashboard assets versioned by timestamp
- support restore and local replay for regression triage

## Canonical dataset

Primary files generated by CI:

- `benchmarks/reports/latest.json`
- `benchmarks/reports/latest.md`
- `benchmarks/reports/history.json`
- `benchmarks/reports/INDEX.md`
- `benchmarks/reports/timeseries.json`
- `benchmarks/reports/timeseries.csv`
- `benchmarks/reports/dashboard.html`
- `benchmarks/reports/benchmark-report-YYYYMMDD-HHMMSS.{json,md}`

Recommended remote layout:

- `benchmarks/<repo>/<branch>/reports/...`
- `benchmarks/<repo>/<branch>/archive/benchmark-report-YYYYMMDD-HHMMSS.{json,md}`
- `benchmarks/<repo>/<branch>/dashboard/index.html`

Use `main` as the default branch namespace for long-term trend baselines.

## Upload workflow

1. Run `.github/workflows/performance.yml`.
2. Download artifact `benchmark-dashboard-publish` and `benchmark-reports` if needed.
3. Sync to object storage.
4. Keep immutable timestamped report files; overwrite only `latest.*`, `history.json`, `INDEX.md`, `timeseries.*`, and dashboard files.

Automated template:

- `.github/workflows/benchmark-storage-sync.yml` provides a manual dispatch workflow template that can:
  - regenerate benchmark reports in CI
  - sync `benchmarks/reports` to S3, GCS, or Azure Blob
- configure provider secrets before use using `BENCHMARK_STORAGE_SECRETS_CHECKLIST.md`.

## Provider examples

S3:

```bash
aws s3 sync benchmarks/reports s3://<bucket>/benchmarks/<repo>/main/reports --delete
```

GCS:

```bash
gcloud storage rsync --recursive --delete-unmatched-destination-objects benchmarks/reports gs://<bucket>/benchmarks/<repo>/main/reports
```

Azure Blob:

```bash
az storage blob sync --container <container> --source benchmarks/reports --destination benchmarks/<repo>/main/reports --delete-destination true
```

## Retention policy

- keep at least 90 days for trend fidelity
- keep at least 100 timestamped runs for noisy environments
- prune oldest timestamped files only after regenerated `history.json` and `INDEX.md` are uploaded

## Restore workflow

To restore trend context locally:

1. Sync remote `reports/` prefix to local `benchmarks/reports/`.
2. Run:

```bash
python3 scripts/load/maintain_benchmark_history.py
python3 scripts/load/build_benchmark_dashboard.py
python3 scripts/load/check_benchmark_relative_regression.py
```

This rebuilds index/timeseries and validates regression gates against restored history.

## CI integration recommendation

- keep CI as source of truth for report generation
- use a follow-up deploy step (or separate workflow) to sync `benchmarks/reports/` to storage
- gate deploy on successful benchmark and threshold checks
