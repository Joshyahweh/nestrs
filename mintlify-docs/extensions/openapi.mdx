---
title: "OpenAPI: auto-generated docs and Swagger UI serving"
description: "Serve an OpenAPI 3.1 document and Swagger UI from nestrs. Enable the openapi feature and call .enable_openapi() to activate /openapi.json and /docs."
---

The `nestrs-openapi` crate discovers every HTTP route registered through `#[routes]` or `impl_routes!` and builds an OpenAPI 3.1 document from them automatically. You get `GET /openapi.json` and `GET /docs` (Swagger UI) with no extra configuration. For request/response schemas and security schemes, you supply the JSON manually or merge fragments from `utoipa` or a codegen step.

## Enable the feature

```toml Cargo.toml
[dependencies]
nestrs = { version = "0.3.8", features = ["openapi"] }
tokio = { version = "1", features = ["macros", "rt-multi-thread"] }
```

## Minimal setup

Call `.enable_openapi()` on `NestApplication` before `.listen()`:

```rust src/main.rs
use nestrs::prelude::*;

#[module]
struct AppModule;

#[tokio::main]
async fn main() {
    NestFactory::create::<AppModule>()
        .enable_openapi()   // serves GET /openapi.json and GET /docs
        .listen(3000)
        .await;
}
```

After booting, open `http://localhost:3000/docs` for the interactive Swagger UI or fetch `http://localhost:3000/openapi.json` for the raw document.

## What nestrs generates

| Area | Status |
|------|--------|
| Paths and HTTP methods | **Yes** — from the route registry |
| Swagger UI | **Yes** — static HTML with CDN `swagger-ui-dist` |
| Operation `operationId` and `summary` | Heuristic by default; override with `#[openapi(summary = "...")]` |
| `tags` | Heuristic from the path; override with `#[openapi(tag = "...")]` |
| `responses` | Default `200 OK`; override with `#[openapi(responses = ((404, "Not found"), ...))]` |
| Request/response schemas from DTOs | **Not generated** — supply manually via `OpenApiOptions.components` or merge `utoipa` |
| `components`, `security`, `servers` | **Optional** — set on `OpenApiOptions` |

## Per-route metadata with `#[openapi]`

Override the auto-generated `operationId`, summary, tags, and response codes on individual handlers:

```rust src/users/users_controller.rs
use nestrs::prelude::*;

#[controller(prefix = "/users")]
struct UsersController;

#[routes]
impl UsersController {
    #[get("/:id")]
    #[openapi(
        summary = "Fetch a user by ID",
        tag = "users",
        responses = ((200, "User found"), (404, "User not found"))
    )]
    async fn get_user() -> &'static str {
        "user"
    }
}
```

## Advanced options

Replace `.enable_openapi()` with `.enable_openapi_with_options(OpenApiOptions { ... })` to configure the document title, servers, components, and security schemes.

```rust src/main.rs
use nestrs::prelude::*;
use nestrs_openapi::OpenApiOptions;
use serde_json::json;

#[tokio::main]
async fn main() {
    let openapi_options = OpenApiOptions {
        title: "My API".into(),
        version: "1.0.0".into(),
        json_path: "/openapi.json".into(),
        docs_path: "/docs".into(),
        api_prefix: "/api/v1".into(),
        servers: Some(vec![json!({
            "url": "https://api.example.com",
            "description": "Production"
        })]),
        components: Some(json!({
            "securitySchemes": {
                "bearerAuth": {
                    "type": "http",
                    "scheme": "bearer",
                    "bearerFormat": "JWT"
                }
            }
        })),
        security: Some(vec![json!({ "bearerAuth": [] })]),
        infer_route_security_from_roles: false,
        roles_security_scheme: "bearerAuth".into(),
        ..Default::default()
    };

    NestFactory::create::<AppModule>()
        .enable_openapi_with_options(openapi_options)
        .listen(3000)
        .await;
}
```

<Note>
  Set `api_prefix` to match your global prefix and URI version (for example `/api/v1`) so documented paths align with real URLs.
</Note>

## Per-route security from `#[roles]`

When you decorate handlers with `#[roles("admin")]`, nestrs registers `roles` metadata in `MetadataRegistry`. Set `infer_route_security_from_roles: true` and `roles_security_scheme` to the name of a scheme you declared in `components.securitySchemes`, and only those operations will include a `security` array in the OpenAPI document.

```rust
OpenApiOptions {
    components: Some(json!({
        "securitySchemes": {
            "bearerAuth": { "type": "http", "scheme": "bearer", "bearerFormat": "JWT" }
        }
    })),
    security: None,                           // no global security requirement
    infer_route_security_from_roles: true,    // only routes with #[roles] get a security entry
    roles_security_scheme: "bearerAuth".into(),
    ..Default::default()
}
```

<Warning>
  This is a heuristic: it keys off `roles` metadata, not runtime guard types. Guards that do not set `roles` metadata will not trigger the hint. Add `#[roles(...)]` or `#[set_metadata("roles", "...")]` for documentation purposes on those handlers.
</Warning>

## Adding schemas with `utoipa`

`utoipa` can derive `OpenApi` components from Rust types. The typical workflow:

<Steps>
  <Step title="Derive schemas on your DTOs">
    ```rust src/users/dto.rs
    use utoipa::ToSchema;

    #[derive(serde::Serialize, serde::Deserialize, ToSchema)]
    pub struct UserDto {
        pub id: String,
        pub email: String,
    }
    ```
  </Step>
  <Step title="Build a utoipa fragment">
    ```rust src/openapi_fragments.rs
    use utoipa::OpenApi;

    #[derive(OpenApi)]
    #[openapi(components(schemas(crate::users::dto::UserDto)))]
    pub struct ApiDoc;
    ```
  </Step>
  <Step title="Merge into OpenApiOptions.components">
    ```rust src/main.rs
    use serde_json::Value;
    use utoipa::OpenApi;

    let utoipa_doc = ApiDoc::openapi();
    let components: Value = serde_json::to_value(utoipa_doc.components).unwrap_or_default();

    let openapi_options = nestrs_openapi::OpenApiOptions {
        components: Some(components),
        ..Default::default()
    };
    ```
  </Step>
</Steps>

## Troubleshooting

| Issue | What to check |
|-------|---------------|
| `/docs` or `/openapi.json` returns 404 | Confirm `features = ["openapi"]` on `nestrs` and that you called `.enable_openapi()` before `.listen()` |
| Routes missing from the document | Handlers must use `#[routes]` or `impl_routes!` to register in `RouteRegistry` |
| Schemas are empty | Core does not infer DTOs — populate `OpenApiOptions.components` or merge a `utoipa` fragment |
| `security` not appearing per route | Set `infer_route_security_from_roles: true` and add `#[roles]` or `#[set_metadata("roles", "...")]` to the relevant handlers |
