---
title: "WebSockets: real-time gateways and JSON event routing"
description: "Add WebSocket endpoints to nestrs with the ws feature. Route JSON frames using #[ws_gateway] and #[ws_routes], and attach guards and pipes per gateway."
---

The `nestrs-ws` crate gives you an Axum-native WebSocket gateway that works with nestrs's DI container. Frames follow a simple JSON contract — `{ "event": "...", "data": ... }` — and the `#[ws_gateway]` and `#[ws_routes]` macros generate the dispatch code so each method on your struct handles one event name. Guards, pipes, and interceptors attach the same way they do for HTTP routes, though the pipeline ordering is different.

## Enable the feature

```toml Cargo.toml
[dependencies]
nestrs = { version = "0.3.8", features = ["ws"] }
tokio = { version = "1", features = ["macros", "rt-multi-thread"] }
serde = { version = "1", features = ["derive"] }
serde_json = "1"
```

## Define a gateway

Apply `#[ws_gateway]` to the struct and `#[ws_routes]` to the `impl` block. Use `#[subscribe_message("event-name")]` on each handler method.

```rust src/chat/chat_gateway.rs
use nestrs::prelude::*;
use nestrs::ws::{WsClient, WsGateway};

#[derive(serde::Deserialize)]
struct ChatMessage {
    text: String,
}

#[ws_gateway]
#[injectable]
struct ChatGateway;

#[ws_routes]
impl ChatGateway {
    #[subscribe_message("chat")]
    async fn on_chat(&self, client: WsClient, msg: ChatMessage) {
        let _ = client.emit("chat", serde_json::json!({ "echo": msg.text }));
    }

    #[subscribe_message("ping")]
    async fn on_ping(&self, client: WsClient, _payload: serde_json::Value) {
        let _ = client.emit("pong", serde_json::Value::Null);
    }
}
```

## Mount the gateway

Register the gateway as a provider and merge the WebSocket route into your app:

```rust src/main.rs
use nestrs::prelude::*;
use nestrs::ws::ws_route;
use std::sync::Arc;

#[module(providers = [ChatGateway])]
struct AppModule;

#[tokio::main]
async fn main() {
    // Build the gateway outside DI for the ws_route closure, or resolve from the module.
    let gateway = Arc::new(ChatGateway);
    let ws = ws_route(gateway).route("/ws");

    NestFactory::create::<AppModule>()
        .use_global_layer(move |router| router.merge(ws.clone()))
        .listen(3000)
        .await;
}
```

Clients connect to `ws://localhost:3000/ws` and send frames shaped as `{ "event": "chat", "data": { "text": "hello" } }`.

## Cross-cutting concerns

Attach guards, interceptors, and pipes to the gateway struct using the `ws`-prefixed decorator macros:

```rust src/chat/chat_gateway.rs
#[ws_gateway]
#[injectable]
#[use_ws_guards(WsAuthGuard)]
#[use_ws_interceptors(WsLoggingInterceptor)]
#[use_ws_pipes(WsValidationPipe)]
struct ChatGateway;
```

These types implement `WsCanActivate`, `WsIncomingInterceptor`, and `WsPipeTransform` from `nestrs::ws`.

<Warning>
  The WebSocket pipeline runs in the order **interceptors → guards → pipes → handler**, the same as microservice handlers. This differs from the HTTP pipeline order. Do not assume cross-cutting types reorder identically between HTTP and WebSocket contexts.
</Warning>

## Errors and exception filters

WebSocket frames are not Axum HTTP responses. `NestApplication::use_global_exception_filter` does **not** run on WebSocket JSON frames.

Server-side failures are emitted to the client on the `"error"` event (`WS_ERROR_EVENT`). Each error shape is documented in the `nestrs-ws` crate-level docs and includes errors for:

- Failed guard checks
- Pipe transform rejections
- Unknown events
- Bad DTO deserialization
- Wire parse errors

To normalize error behavior across your gateway, share guard and pipe types or wrap `WsGateway::on_message` in a thin adapter.

## Connection handshake and headers

Every `WsClient` carries a `WsHandshake` with the headers from the upgrade request. Use `client.handshake()` to access them for auth or routing decisions inside handlers:

```rust
#[subscribe_message("secure")]
async fn on_secure(&self, client: WsClient, payload: serde_json::Value) {
    let token = client
        .handshake()
        .headers()
        .get("authorization")
        .and_then(|v| v.to_str().ok());
    // validate token...
}
```

## Manual `WsGateway` implementation

If you need full control, implement `WsGateway` directly without the macros:

```rust src/raw_gateway.rs
use async_trait::async_trait;
use nestrs::ws::{ws_route, WsClient, WsGateway};
use std::sync::Arc;

struct EchoGateway;

#[async_trait]
impl WsGateway for EchoGateway {
    async fn on_message(&self, client: WsClient, event: &str, payload: serde_json::Value) {
        let _ = client.emit(event, payload);
    }
}

fn echo_route() -> axum::Router {
    let gateway = Arc::new(EchoGateway);
    ws_route(gateway)
}
```

<Note>
  The `ws` feature exposes the `nestrs-ws` crate as `nestrs::ws` in your code. For Socket.IO-style clients, see the [`socketioxide`](https://crates.io/crates/socketioxide) crate or maintain the `{ "event", "data" }` JSON contract documented in `nestrs-ws`.
</Note>
